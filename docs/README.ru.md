Codex, Claude Code и opencode хранят свои эндпоинты каждый в своём файле, в
своём формате и под своим названием для одного и того же. Чтобы добавить
провайдера или переключиться между двумя, приходится руками открывать три файла.
ConfAI делает это одной командой и никогда не переформатирует то, чего не менял.

## Установка

Linux и macOS:

```sh
curl -fsSL https://github.com/redstone-md/ConfAI/releases/latest/download/install.sh | sh
```

Windows:

```powershell
irm https://github.com/redstone-md/ConfAI/releases/latest/download/install.ps1 | iex
```

Оба скрипта определяют платформу, скачивают подходящий архив релиза и
`SHA256SUMS`, сверяют контрольную сумму и только после этого кладут бинарник на
место. Пайп скрипта из интернета в шелл — вопрос доверия, который решать вам; в
[INSTALL.md](../INSTALL.md) показано, как сначала его прочитать.

Через cargo — но только после того, как крейт будет опубликован на crates.io; на
момент v0.0.1 он не опубликован, так что эти две команды пока не сработают:

```sh
cargo install confai --locked    # собирает из исходников, нужен Rust 1.88+
cargo binstall confai            # вместо сборки скачивает архив релиза
```

Либо вручную: возьмите архив под свою платформу со
[страницы релизов](https://github.com/redstone-md/ConfAI/releases/latest),
сверьте его с опубликованным рядом `SHA256SUMS` и положите бинарник в `PATH`.

Остальное — в [INSTALL.md](../INSTALL.md): все платформы, флаги установщиков, что
происходит с `PATH` и как удалить.

## Что он делает

```
$ confai list
agent        detected         providers  active  model          config
Codex        binary + config  3          primary gpt-5.6-terra  ~/.codex/config.toml
Claude Code  binary + config  1          byesu   opus[1m]       ~/.claude/settings.json
opencode     binary + config  11         vendor              ~/.config/opencode/opencode.json
```

- Одна команда переключает всех агентов, у которых есть такой эндпоинт:
  `confai provider use primary`.
- Один пресет прописывает один и тот же эндпоинт во все: `confai preset apply byesu --all --use`.
- `confai provider sync` подтягивает список моделей, которые эндпоинт реально
  отдаёт, вместе с лимитами контекста и вывода.
- Комментарии, порядок ключей и незнакомые ключи переживают правку. Перед каждой
  записью делается бэкап, а `confai undo` возвращает всё назад.

Запуск `confai` без аргументов открывает двухпанельный браузер: агенты слева,
эндпоинты выбранного агента справа.

<details>
<summary><b>Командная строка</b> — все подкоманды и флаги</summary>

```sh
confai                                    # интерактивный режим
confai list                               # что установлено и где лежит
confai provider list --check              # все эндпоинты и отвечают ли они
confai provider add byesu \
    --agent codex \
    --base-url https://byesu.com/v1 \
    --api-key "$BYESU_API_KEY" \
    --wire-api chat --use
confai provider use primary               # переключить всех, у кого он есть
confai provider sync vendor --prune       # забрать список моделей у эндпоинта
confai preset apply byesu --all --use     # один эндпоинт, все агенты
confai doctor                             # всё ли ещё парсится и разрешается
confai undo                               # вернуть как было
```

`--agent` работает с одним агентом, `--all` — со всеми установленными. Без них
читающие команды охватывают всё, а пишущие просят выбрать.

| Команда | |
|---|---|
| `list` | какие агенты установлены и куда они направлены |
| `provider list` | эндпоинты выбранных агентов; `--check` опрашивает каждый |
| `provider add <id>` | добавить эндпоинт или изменить у существующего те поля, что переданы |
| `provider remove <id>` | удалить эндпоинт |
| `provider use <id>` | направить агента через один из его эндпоинтов |
| `provider check [id]` | спросить эндпоинты, живы ли они и что отдают |
| `provider models [id]` | что отдаёт эндпоинт, с лимитами и ценами |
| `provider sync <id>` | записать список моделей в конфиг |
| `preset list` · `preset show <id>` | какие есть пресеты и что именно пресет запишет |
| `preset apply <id>` | записать эндпоинт из пресета выбранным агентам |
| `model [model]` | показать или задать модель агента |
| `path` · `edit` | путь к конфигу агента · открыть его в `$EDITOR` |
| `doctor` | проверить, что все конфиги парсятся и все указанные провайдеры разрешаются |
| `about` · `update` | версия и где лежит состояние · есть ли релиз новее |
| `undo` | восстановить конфиг из бэкапа, сделанного перед последней записью |

`provider add` принимает `--base-url`, `--api-key`, `--wire-api` (`chat`,
`responses` или `anthropic`), `--name`, повторяемый `--set KEY=VALUE` для
специфичных для бэкенда ключей, а также `--use` / `--sync` — выбрать эндпоинт и
подтянуть его модели сразу после записи. У `provider check` есть `--timeout` в
секундах, по умолчанию 10. У `provider models` — `--select <model>` и
`--refresh`. У `provider sync` — `--prune`, `--dry-run` и `--refresh`. У
`preset apply` — `--api-key`, `--use` и `--sync`.

`list`, `doctor`, `about`, `update` и `preset list` не принимают выбор агента —
они всегда охватывают всё.

</details>

<details>
<summary><b>Интерактивный режим</b> — палитра команд, карточка эндпоинта, полная раскладка клавиш</summary>

Палитра команд на `Ctrl+P` показывает все действия вместе с клавишами, которыми
они запускаются, — так сочетания запоминаются по ходу работы, а не с этой
страницы:

<p align="center">
  <img src="../assets/screenshots/palette.png" alt="Палитра команд со списком всех действий и их клавиш" width="900">
</p>

`Enter` на эндпоинте показывает всё, что о нём известно, включая список моделей с
лимитами контекста и вывода:

<p align="center">
  <img src="../assets/screenshots/detail.png" alt="Карточка провайдера: поля эндпоинта и список моделей" width="900">
</p>

| клавиша | |
|---|---|
| `Ctrl+P` / `Ctrl+K` | палитра команд — все действия, с поиском |
| `↑` `↓` / `k` `j` | перемещение · `Tab` `←` `→` смена панели |
| `Enter` | карточка эндпоинта вместе со списком моделей |
| `/` или `Ctrl+F` | фильтр эндпоинтов по id, хосту или модели |
| `u` | направить агента через выбранный эндпоинт |
| `m` | выбрать модель для этого агента |
| `a` `e` `d` | добавить · изменить · удалить |
| `c` / `C` | проверить этот эндпоинт · проверить все |
| `s` / `S` | синхронизировать модели · синхронизировать с удалением устаревших |
| `p` | применить пресет |
| `?` | о программе и полная раскладка клавиш |
| `r` `q` | перечитать с диска · выход |

Мышь работает: клик выделяет, повторный клик открывает, колесо прокручивает, клик
по подсказке запускает действие.

Клавиши определяются по физическому положению, поэтому продолжают работать на
нелатинской раскладке: `й` — это `q`, `Ы` — это `S`. У `/` на кириллице нет
эквивалента по положению, поэтому фильтр открывается ещё и на `Ctrl+F`.

Правки идут по тому же пути «загрузить — изменить — сохранить», что и в CLI, так
что все гарантии для ваших файлов остаются в силе.

</details>

<details>
<summary><b>Чего он не сделает с вашими файлами</b> — комментарии, порядок ключей, бэкапы</summary>

Конфиги пишут руками, а в написанных руками файлах есть вещи, которые наивная
пересериализация уничтожает.

- **Комментарии остаются.** Конфиги Codex правятся через `toml_edit`, так что
  запасной эндпоинт, отложенный в закомментированный `base_url`, никуда не
  денется.
- **Меняется только то, что вы поменяли.** Порядок ключей, отступы и незнакомые
  ключи остаются нетронутыми: каждый бэкенд правит разобранный документ на месте,
  а не пересобирает файл по своему представлению о нём.
- **JSON с комментариями отвергается, а не калечится.** ConfAI пришлось бы их
  выбросить, поэтому он останавливается и говорит об этом.
- **Перед каждой записью делается бэкап** рядом с оригиналом, под именем
  `<name>.confai.bak`, а сам файл заменяется атомарно. `confai undo`
  восстанавливает его.

</details>

<details>
<summary><b>Модели и доступность</b> — откуда берётся список моделей и что делает --prune</summary>

opencode не предложит модель, о которой ему не сказали, и ему нужен явно
указанный лимит контекста. `confai provider sync <id>` обращается к `/v1/models`
эндпоинта, ищет каждый id на [models.dev](https://models.dev), чтобы узнать
лимиты контекста и вывода, и записывает результат — не трогая `variants` и всё
остальное, что вы настроили. Каталог кешируется на сутки; `--refresh` скачивает
его заново.

Синхронизация — это слияние, поэтому модель, которую шлюз уже снял с раздачи,
останется в конфиге, пока вы не скажете иначе. `--prune` убирает те, что эндпоинт
больше не отдаёт, и переводит выбор модели на одну из оставшихся, если удалил
выбранную:

```sh
confai provider sync vendor --prune --dry-run   # посмотреть, что уйдёт
confai provider sync vendor --prune
```

В интерактивном режиме `s` синхронизирует, `S` — синхронизирует с удалением.

`confai provider models <id>` показывает, что отдаёт эндпоинт, ничего не
записывая, а `--select` делает одну из моделей моделью агента. Это работает и для
Codex с Claude Code, которые хранят модель, но не список моделей.

`confai provider check` — тот же запрос без записи: сообщает, поднят ли каждый
эндпоинт, как быстро он ответил и сколько моделей отдаёт.

</details>

<details>
<summary><b>Пресеты</b> — один рецепт эндпоинта для любого агента</summary>

Пресет — это один эндпоинт, описанный один раз в терминах, не привязанных к
агенту, чтобы один и тот же рецепт применялся к любому из них:

```sh
confai preset list
confai preset show byesu
confai preset apply byesu --all --api-key sk-... --use --sync
```

Двадцать шесть встроенных пресетов лежат в [`presets/`](../presets/) — по одному
TOML-файлу на каждый, вшиваются в бинарник на этапе сборки — и покрывают OpenCode
Zen, OpenRouter, OpenAI, Anthropic, Groq, xAI, Mistral, Cerebras, Together,
DeepSeek, DeepInfra, Fireworks, Moonshot, Z.ai, Chutes, Baseten, Vercel AI
Gateway, Venice, Novita, Byesu, Ollama и LM Studio. Добавить свой — это
пул-реквест ровно с одним новым файлом. Собственные пресеты кладутся в
`~/.confai/presets/` и перекрывают встроенный с тем же id.

</details>

<details>
<summary><b>Агенты</b> — три схемы конфигов и что ConfAI делает с каждой</summary>

| Агент | Конфиг | Ключи | Именованные провайдеры | Список моделей | Переключение |
|---|---|---|---|---|---|
| Codex | `~/.codex/config.toml` | тот же файл | да | нет | `model_provider` |
| Claude Code | `~/.claude/settings.json` | блок `env` | через ConfAI | нет | `ANTHROPIC_*` |
| opencode | `~/.config/opencode/opencode.json` | `~/.local/share/opencode/auth.json` | да | да | `provider/model` |

`CODEX_HOME`, `CLAUDE_CONFIG_DIR`, `OPENCODE_CONFIG` и `XDG_CONFIG_HOME`
учитываются — ровно так же, как их учитывают сами агенты.

Claude Code смотрит на один эндпоинт за раз, через переменные окружения в своих
настройках, и ему негде хранить те эндпоинты, которыми вы сейчас не пользуетесь.
ConfAI держит этот список в `~/.confai/agents/claude.json` и пишет в файл,
которым владеет Claude Code, только выбранную запись.

opencode разнесён по двум файлам: провайдеры в `opencode.json`, ключи в
`~/.local/share/opencode/auth.json`, куда их кладёт `opencode auth login`. ConfAI
читает оба, поэтому проверка доступности уходит с тем ключом, который реально
использовал бы opencode, а не возвращает ложный 401. Новый ключ пишется в
`auth.json`; ключ, уже лежащий прямо в `opencode.json`, обновляется там же, где
лежит, — потому что молча переносить секрет из файла в файл само по себе
неприятный сюрприз. OAuth-сессию в `auth.json` ConfAI показывает, но никогда не
перезаписывает: он предложит выполнить `opencode auth logout`, а не завершит её
втихую.

Добавить агента — это один файл в `src/agent/`, реализующий `Agent` и
`AgentConfig`; ничто выше этого слоя не знает, с каким агентом оно говорит.

</details>

<details>
<summary><b>Обновления</b> — как работает проверка и как её отключить</summary>

`confai update` сообщает, есть ли релиз новее, кратко пересказывает изменения и
печатает, как обновиться.

В повседневной работе спрашивать не нужно. После команды ConfAI печатает в stderr
две строки, если вышел релиз новее:

```
◆ 0.0.1 → 0.0.2 available
  · provider sync now prunes retired models
  · run `confai update` for the rest
```

Это уведомление рисуется из кеша, который обновляется не чаще раза в сутки, так
что обычный запуск ничего не стоит; а когда кеш устарел, у проверки есть
четыреста миллисекунд на ответ, после чего запуск махнёт рукой и попробует
завтра. После неудачной проверки следующая будет не раньше чем через час, а не на
каждом вызове. Переменная `CONFAI_NO_UPDATE_CHECK` отключает всё это целиком.

ConfAI не подменяет собственный бинарник. `cargo` и установочные скрипты уже
делают это как надо, а инструмент, который переписывает сам себя, держа при этом
ваши ключи, — обмен куда хуже, чем одна напечатанная строка.

</details>

<details>
<summary><b>Как поучаствовать</b> — добавить пресет или агента</summary>

Пресет — это один новый файл в `presets/`. Новый агент — один новый файл в
`src/agent/`, реализующий `Agent` и `AgentConfig`; слои выше остаются
нетронутыми. Перед пул-реквестом прогоните `cargo test` и
`cargo clippy --lib --bins --tests`. См. [CONTRIBUTING.md](../CONTRIBUTING.md).

</details>
